今日最有感的事

Higgsfield 把「Supercomputer Skill」串進 Claude——AI 從生成工具走向內容 Agent

今天看到 @kufutw（顯示名「AI Threads」）在 Threads 發了一串 4 則的串文，講 Higgsfield 把「Supercomputer Skill」正式整合進 Claude。整則貼文 12 小時內 7,701 次瀏覽、主貼 125 讚 / 13 轉發 / 155 分享，外部連到 Higgsfield 官方在 X 的公告。

它在說一件事：以前要拆五個工具才做得完的短影音生產流程，現在丟一個影片連結給 Claude，它就會自己做完。Claude 拿到影片後，會分析鏡頭、節奏、hook、運鏡、剪輯邏輯，然後在同一個對話視窗裡直接生新素材。背後並不是單一模型——Higgsfield MCP 接了 30+ AI 模型（Kling、Veo、Seedance、Flux、Minimax Hailuo 等），由 Claude 自己挑模型、排 workflow、產素材。官方甚至展示了一條完整管線：分析 TikTok/Reels 廣告 → 生腳本 → 生 AI UGC → 做影片 → 自動產出整套廣告素材。

@kufutw 自己給的結論是：「這代表 AI 正從『生成工具』進化成『內容 Agent』。」這句話大致同意，但要降溫看。

真正值得抓出來的訊號有兩個：

第一個是 MCP 在多模型編排上的角色越來越明確。一年前大家還在爭「哪一個影片模型最強」，現在不爭了——直接 MCP 串一層，把 30+ 模型當 backend，由 Claude 當 orchestrator。這跟我自己日常用 Claude Code 的體感一致：我現在很少在「選工具」，而是在「描述要做的事」，剩下交給 Agent 自己拆解、自己挑工具。Higgsfield 這次的整合，就是把這套體感從 coding 場景複製到「短影音生產」場景。對非工程師讀者補一句：MCP（Model Context Protocol）是 Anthropic 推的一個標準，讓 Claude 這類模型可以用統一介面接外部工具，所以「Claude 同時管 30+ 個影片模型」在工程上才會變成一件相對輕量的事。

第二個是「分析爆款 → 自動生成同類素材」這條鏈被打通了。以前分析爆款是創作者的 know-how，要花時間看、抄、再調整。現在 Claude 直接「理解什麼影片會爆」（@kufutw 的原話），然後接後端模型生新素材。更直白地說：創作者的稀缺性從「會分析」往「會品味」位移——AI 能幫你拆完一支爆款的所有要素，但你要不要那種味道、要不要 follow 那條爆款邏輯，這個選擇還在人身上。

要降溫的地方也很明確：貼文裡「最可怕的是」「這已經不是 AI 生影片」這類句型都是行銷渲染，實際生成品質、模型挑選邏輯透不透明、單次成本、產出風格能不能精準控制——這些都沒有被展示。對我這種非短影音創作者來說，這則更像是「讓我認識 MCP 在內容產業的擴張節奏」而不是「我明天就要用這套」。

但我把它放進了長期觀察清單：當 Claude 開始能接到外部產業的 30+ 模型後端時，醫療場景什麼時候會輪到？這是我自己 ECG 與急診工具鏈下一階段會碰到的問題。
