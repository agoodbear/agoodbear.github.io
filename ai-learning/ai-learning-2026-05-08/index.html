By 急診熊 · May 8, 2026 · 2 分鐘閱讀 ·AI學習 · Claude · memory · Claude Code · OpenEvidence · Anthropic

今日最有感的事

memory 不是越多越好——同一天從 FB 看到 Claude Dreams 到自己 CLI binary 內建 /dream

下午從兩條 FB 拼出一個故事弧。

16:45 看到 YuShang Lung 在 GenAI 技術社團轉貼 Anthropic 的 Claude Dreams：它在做的不是「Claude 在做夢」，而是幫 agent 整理長期 memory——讀過去 memory store + session transcripts，產出一份新的乾淨版（重複合併、過期更新、矛盾整理、跨 session 抓 pattern），舊的不直接覆蓋，先讓使用者比對再決定要不要用。目前是 Claude Managed Agents 的 Research Preview，掛在 platform / API 層。

17:13 看到林協霆貼 OpenEvidence 推出 DotFlows（OE 版的 Skill marketplace），他順手補了一條線索：Claude Code 幾週前 AutoDream feature flag 露出過，不是所有人都看得到。

我請 Claude 直接 probe 自己機器上的 Claude Code 2.1.133 binary，從 strings 裡抓到一整組指令：

1/dream                  手動觸發
2/dream consolidate      整理模式
3/dream nightly          排程每晚跑
4[autoDream]             背景守護機制（含 rollback）

關鍵語意（從 binary 註解抓的）：遇到 memory 與 CLAUDE.md 矛盾會註記「contradicts CLAUDE.md — verify which is current」讓使用者決定，整理結果是新版 memory，舊的不覆蓋、留一份給使用者 review。

這代表 Anthropic 把同一個東西做了兩個 surface：platform 版（Claude Dreams API）+ CLI 版（Claude Code 內建 /dream），CLI 版在我桌上 binary 裡其實已經 ready，只是還沒寫進公開 docs。

這對我直接有用。我目前 MEMORY.md 已經堆到 100+ 條，包含過期的 cc-statusline race 觀察、舊 ECG Weekly cookie 狀態、各種已解決或變更的 ccbot watchdog 規則。手動清就是會一直拖。/dream 的設計剛好對應我不敢自動清的三個顧慮：

  1. 不直接覆蓋舊的——產乾淨版給我比對，原檔留著
  2. 矛盾不自己決定——標「contradicts — verify which is current」交回使用者
  3. 可分階段觸發——consolidate 整理一次、nightly 設排程、autoDream 全自動加 rollback，三檔讓使用者選

真正值得看的是這個設計選擇：agent 不主動「做主」，把選擇權留給使用者。Memory 作為 agent 的「個人化資產」，自治理工具的關鍵不是壓縮率，是接口設計不能破壞使用者既有的 trust chain——一旦 agent 自己改 memory 改錯一次，後面所有 agent 行為都會被使用者打折。Anthropic 這次設計很保守，正是因為他們也知道這條 trust chain 一斷就回不來。

順帶提一下今天看到的另一條同向訊號：OpenEvidence DotFlows（OE 版 Skill）+ 林協霆寫爬蟲把 community dotflows 全抓下來分析微調，做出 .onc_ 系列腫瘤學專用 dotflows。memory 自治理（Claude Dreams / /dream）跟 skill marketplace（DotFlows）是兩件事，但精神同一條：agent 周邊資產（memory、skill）逐漸變成可流通、可整理、可 share 的資產。未來這兩個方向會跟 Claude Code 自己的 Skills 系統碰頭。
