AI 學習

2026-04-29 AI 學習日誌

今天三件 AI 觀察一條線：GPT 中文標準字 prompt 已能直接抄、Firecrawl /parse API 對已建本機 pipeline 的我邊際效益低先觀望、5/11 賴賢勇『生成式 AI 協助論文撰寫』與 5/26 WOS AI x Scite AI x Endnote 工作坊兩堂院內課程要排優先序——AI 工具洪流裡，分流『要直接用 / 先觀望 / 要學』比『跟新聞跑』重要。

By 急診熊 · Apr 29, 2026 · 2 分鐘閱讀 ·AI學習 · 中文字設計 · Firecrawl · AI 課程選擇 · Mail Digest

今日最有感的事

今天從早上 10 點半看到黃祺浩 FB 那篇「GPT 做中文標準字設計」開始，就感覺到一條主線：面對 AI 工具與訊息洪流，重點不是跟新聞跑，是把每一個進來的東西分流到「要直接用 / 先觀望 / 要學」三個籃子，再決定要花多少時間。

第一籃「直接抄就能用」是 GPT 中文標準字 prompt。黃祺浩給出一份七項規格的完整 prompt（筆畫厚實圓潤、末端誇張肥大、可減筆連筆、扁平字體、白底、高解析），新版 GPT 對中文字支援好到可以拿這個 prompt 跑出可用的品牌標準字設計。今天沒馬上動手（要先把週報寫完），但已經寫進 todo-list「好玩的專案」section（2026-04-29），標記之後找一天用 /codex-image Mode 2 跑「急診熊心聲 / 急專大補帖 / 急診週報」三組標準字 + 4× 放大列印級。值得記住的不是這條 prompt 本身（會過時），是『遇到一個不錯的 prompt，先存進 todo-list 不要硬塞進今天』的這個動作——這跟昨天的 Context Mode 走同一條鐵律：今天眼睛被新東西吸引不代表今天就要動手，把它寫進系統等之後有空檔再做才不會打亂主軸。

第二籃「先觀望」是 Firecrawl /parse API。bing_sunzhi 在 Threads 推：PDF / DOCX / XLSX → 乾淨 Markdown，Rust 重寫號稱快 5 倍、表格保留好、Zero Data Retention（不留資料）。表面看就是命中我那 35 本急診教科書、Zotero papers、Kindle 戰場。但仔細看自己的現況：那些戰場早就走過本機 pipeline + BGE-M3 一輪——Zotero 1,212 篇用 opendataloader-pdf 跑完了、複雜表格 / 公式 / 雙欄 PDF 有 pdf-figure-extractor、Kindle 那些 reflow 不支援的書是 DRM blob 不是 PDF（/parse 救不了）。再加上 mcp__firecrawl__* 已經在 plugin list 裡，要用隨時可叫。結論是：對我現況沒有換 stack 的理由，唯一值得記一筆的場景是 exam-rationale 偶爾收到外部寄來的 Word / PDF 考題、表格亂掉時當一次性救火工具。這是昨天看 Context Mode 學到的那條鐵律的延伸——不是工具不好，是對你現有 stack 的邊際效益要算清楚。當你已經有一條跑得通的本機 pipeline，新來的雲端 API 要做的不是「取代」而是「補位」，補位就只在補位的場景裡用，不必整套搬家。

第三籃「要排序去學」是兩個院內 5 月 AI 課程，今天從 Mail Digest 撈出來：5/11（一）大千醫院賴賢勇副院長談「生成式 AI 協助研究論文撰寫與投稿」 vs 5/26（二）蘭陽 WOS AI x Scite AI x Endnote 論文寫作工作坊（限 20 位、4 小時專業課程）。兩堂課題目像但角度不同：賴賢勇講的是「怎麼用 AI 寫論文」（戰術級、具體 workflow），WOS x Scite x Endnote 工作坊是「Web of Science + Scite 引用驗證 + Endnote 工具串接」（工具棧級、跟我已經在用的 OpenEvidence + PubMed + Zotero 是同一套品味）。初步傾向兩堂都報名——5/11 那堂時間短（2.5 小時）拿 take-away 風險低；5/26 那堂限 20 名要快搶，工具串接的東西看完通常需要當場動手。Mail digest 自動把這兩條挑進「🟡 本週處理」桶子是 mail-digest skill triage 規則生效的具體案例（它判讀「AI / 院內專業課程」的關鍵字觸發了高優先標記）。

把三籃合起來看：今天沒裝任何新工具、沒寫任何新 code，但有一個 prompt 進 todo-list、一個 API 被分流到「現階段不急」、兩堂課從信箱撈出來進「本週決定要不要報名」。這種 throughput 看似什麼都沒做，其實是讓未來的自己不會被同樣的訊息再打斷一次——下次再看到 Firecrawl 推銷文，腦袋裡的 mental cache 就會直接 hit「已評估、現階段不裝」，不會再打開連結點開讀完一輪。Mail digest 自動把訊息分桶 + todo-list 自動把 prompt 收編 + 信件 triage 規則精準觸發，這套 workflow 跑了一週左右，今天第一次明確感覺到它在替我擋下「跟新聞跑」的衝動。AI 工具洪流不會停，但你的注意力會被這些自動 triage 機制保護下來，留給真正要動手的事。
