## 今日最有感的事

### Google 也下場做 Agent Skills 了

@tootiredbear 今天丟出來的 `google/skills` repo 一看就懂在做什麼——Google 把自家產品（Google Cloud、Gemini API、BigQuery、Cloud Run）的產品知識、操作流程、最佳實務，整包成 AI Agent 可以調用的技能模組。**這代表 Agent Skills 已經從 Anthropic 一家提出的概念，演變成各大平台廠商的標配做法**。Google 沒有發明新格式，而是用 SKILL.md 同一套規格、把自家最熟的東西打包成 agent 直接能用的能力。

對非工程師讀者補一個背景：Agent Skills 是把「一段觸發詞 + 一份 markdown 說明 + 可選的 scripts」綁成一個技能包，讓 AI agent 在偵測到特定情境時自動掛載對應的技能。原本是 Anthropic 推 Claude Code 時帶出來的概念，現在 Google 把它變成自家產品線的官方延伸。

留言區有人問「能不能放進 Gemini Gem 用？」@tootiredbear 給的判斷很實在：不能直接以 Agent Skill 形式塞進 Gem 執行，但可以把 SKILL.md 整理成 Gem 的自訂指令使用。**真正值得看的是這個轉變方向**——大廠開始用統一的 skill 格式發行自家工具的「使用手冊」，而不是丟一份文件叫 AI 自己讀。對我這種已經累積 50+ 個 Claude Code skill 的人，這份 repo 直接價值不算最高（我不重度用 GCP/BigQuery），但**架構長相值得拿來對照**——Google 怎麼寫 SKILL.md、怎麼設計觸發條件，反過來可以改進我自己 skill 的結構設計。

### 下載的不再是工具，是數位員工

@0xspeter 整理的本週 GitHub 趨勢有一句金句很扎實：「以前大家下載的是工具，現在大家下載的是『數位員工』。」他點出本週六個爆紅軸——科研助手、影片理解、AI 記憶系統、語音生成、Browser Agent、一人團隊工作流，幾乎全在 Agent 化。

裡面他挑了四個 repo 出來：

| 方向 | 專案 | 在解什麼 |
|---|---|---|
| 🧠 長期記憶 | agentmemory | AI 跨 session 有記憶、不再每次失憶 |
| 🎬 影片理解 | NVIDIA video-search-and-summarization | AI 真的「看懂」影片內容 |
| 🕷 Browser Agent | CloakBrowser | 像真人一樣操作網頁 |
| 📈 一人公司 | gstack | 一個人公司時代越來越近 |

**更直白地說**：這四條軸看似分散，其實是同一件事的不同切面——讓 AI 從「能回答問題」變成「能替你完成工作」必須補齊的能力鏈：要記得（memory）、要看得懂（video）、要會操作（browser）、要能串成完整工作流（stack）。

對我這種急診醫師日常的接點：agentmemory 那條最直接——我已經有自己土法煉鋼的記憶層（handover、Roam、Supabase）但都靠手動或 cron 維持；如果有開源解能本機跑（SQLite + all-MiniLM-L6-v2 embeddings、不用 API key）、原生支援 Claude Code，等於可以把「跨 session 自動帶相關 context 進來」這件事從半自動推到全自動。Browser Agent（CloakBrowser）跟我之前在 kobo-extract / books-extract 的場景同源，只是更通用一層。這些都是「下次有空再評估接不接」的清單，不急但要記著。
