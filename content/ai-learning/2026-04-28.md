## 今日最有感的事

昨天那條「敢罵被推爆的工具」的判斷力今天直接被生活演練——上午看到 techtip_s 整理的「Claude Code 神級工具上篇 5 個 GitHub repo」，一打開最猛的 **Context Mode**：10.6k stars、HN #1 570 點、兩個月衝起來、作者活躍、Discord 社群熱絡、宣稱「PostToolUse hook 自動攔截壓縮 tool 輸出，省 98% token」。第一輪反射就是「直接命中我 batch ingest / Playwright / yt-dlp / whisper 大輸出的痛點，要裝」。但點進去 issues 看到 **#368 戳破核心**：作者本人在 comment 承認壓縮是 **opt-in**、Claude（或人）要主動呼叫 `ctx_execute` / `ctx_index` 才會省。Anthropic 的 PostToolUse hook 機制根本不允許事後改寫 tool_result，所以你跑 Bash / Read / 別人的 MCP，輸出還是整包進 context，hook 只是事後 log 到 SQLite 給 session 續接用。**作者自己標籤這是 "docs bug, not a missing feature"**。

這個翻盤的關鍵不是 repo 有沒有偷工，是「README 講的跟 Anthropic SDK 平台允許的不一樣」。**不要看 README 怎麼宣稱，要看物理 / 平台 / 工程量讓不讓**——昨天的 obscura 紅旗（V8 不可能 30 MB、5 分鐘編譯不完）走的是同一個方法論。Context Mode 公司背景 / commit 史完全沒問題，但「README ≠ 實際能力」這條同樣適用。後來改建議走 **MCP-only install**（不裝 hook 版）：Claude 看到大任務時主動呼叫 ctx_* 工具，不會把所有 hook 攔在 workflow 上，session 連續性那塊倒是真有用。

下午被推到一個「跨工具拼裝」的有趣 task：看到 IG 上 Stroke 期刊 May 2026 封面風格很有 editorial 味道（炭筆顆粒+米黃背景+黑色側臉剪影+腦後紅斑塊+大標題），想把自己頭像套這個風格。實作順序是：抓封面圖分析五個風格訊號（顆粒紋理、半透明腦剖面、一抹紅、極簡配色、頂部排版）→ 確認三個自由度（剪影 vs 保留五官、標題文字、紅色元素換不換）→ 拿大頭照當側臉視覺參考、把標題改「擊掌熊」、紅色斑塊換成 **VF 波形**（更貼急診 ECG 主題）→ 丟 Codex CLI `$imagegen` 直式 1024×1536 期刊封面構圖 → 中途撞到 Codex CLI 版本太舊（要 0.125），順手升級重跑。整套流程把「視覺分析 → 自由度釐清 → prompt 翻譯 → 工具升級 → 出圖」串成一條，跟前一天看的吳相勳「邊開車邊讓 Codex 寫 200 頁書」是同一個母題：**LLM 變成了能把領域風格 + 個人內容 + 自由度設計三條線同時拼起來的執行層**。生出來的圖效果不錯，唯一可以再調的是頭髮（Codex 把 taper fade 短髮畫成中長微亂），這是 prompt 還沒收到位的具體訊號。

最後問了一個基礎工事的問題：「session 重啟 context 會清空嗎？」答案分兩塊。會清的是 session 對話本身、剛載入的 deferred tool schema、TodoWrite / inflight task 等 session state；不會清的是 MEMORY.md + 所有 feedback / project / reference 記憶檔、CLAUDE.md、Supabase `handovers` table 最近 3 筆（SessionStart 自動注入）、`~/.claude/handover.db` 完整歷史、inflight checkpoint、todo-list.md 跟 Roam / Zotero / GitHub / Hugo 這些外部系統。**重啟前長任務主動開 `/checkpoint`、session 結束有重要進度就 `/handover`，下次新 session 開起來會看到交接單**。這不是新功能介紹，是把「會話有限、記憶系統無限」的 mental model 講清楚——兩個多月把這套基礎工事建好的回報就是現在這個體感：對話被打斷不會掉重要進度。

把今天三件事串起來看：「先推、再撤」（Context Mode）、「視覺分析→prompt→工具升級拼成一條」（擊掌熊封面）、「會話有限但記憶系統無限」（重啟保留範圍）—— 三個都在同一個方向上：**面對 AI 工具洪流，重要的不是裝得快，而是會問對問題**（README 講的平台讓不讓做？這個自由度該怎麼釐清？這套基礎工事撐不撐得起斷線？）。能問對問題的人，不會被推爆的工具帶著走。
