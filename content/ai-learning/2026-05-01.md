## 今日最有感的事

今天 AI 工具生態打在兩條線：**觀察別人怎麼用** vs. **自己親手 A/B test**——前者是輸入，後者才是真正建立判斷力的地方。

**觀察線之一：Boris Cherny 公開的 15 個 Claude Code 隱藏功能**

早上滑到 aiposthub Threads 整理了 Claude Code 創辦人 Boris Cherny 公開分享的 15 個隱藏用法，作者把核心拉成 7 串重點：AI 從聊天工具變成自動系統是真正分水嶺、`/loop` + `/schedule` 自動排程（5 min 處理 PR / 30 min 整理 Slack / 每小時清舊任務）、不直接用 AI 而是先做成 Skill 再用 loop 自動跑、工作流插「控制點」（自動載入 context、記錄指令、權限請求傳手機核准）、AI 看畫面（Chrome extension 寫 code → 看前端 → 自己修閉環）、手機遠端控制讓工作脫離裝置、真正的問題是「你有沒有把工作交給 AI 跑」。

對照下來自己已實作的有 5/7：28 個 skills、launchd cron（21:00 Hugo+Roam、ECG 週更新、薪資爬蟲、mail digest 等）、CC Bot 手機遠端、handover/checkpoint 雙系統、Discord webhook 控制點。**唯一還沒玩的是「AI 看畫面」的 Chrome extension 閉環**——這是值得排進近期試做清單的方向，因為前端開發的「寫 code → 開瀏覽器看 → 改 code」迴圈如果能讓 AI 自己跑完，整體生產力會再跳一階。

**觀察線之二：王阿達轉的 Open Design（Claude Design 開源逆向版）**

晚上王阿達 FB 轉了一篇王阿達自己寫的文章，主角是 nexu-io 團隊做的開源版「Open Design」，5K stars 起跳，作者 @tuturetom 宣稱還原 Claude Design 95%。最關鍵的差別是**不綁 Anthropic 模型、可本地部署**——支援 Claude Code / Codex / OpenClaw / Cursor / Gemini CLI / Qwen / GitHub Copilot CLI 全通，意思是不用另外開 Claude.ai 視窗，直接在自己現有 Claude Code 工作流跑設計任務。

技術細節值得記住的：內建 19 個設計 skill + 71 套設計系統（Stripe / Apple / Tesla 等品牌風格）、五維自我批判 + P0/P1/P2 checklist + 禁忌黑名單防泛用 emoji 與虛構數據（反 AI-slop 機制）、本地 SQLite + 可攜 Markdown skill 檔——架構跟自己現在 Claude Code skill 系統天然契合。已寫進 todo-list「好玩的專案」分類，等之後有設計需求（ECG 週報 hero / 急專大補帖 landing page 改版）再跑來試，先不貿然占硬碟。

**實踐線（最有感）：第一次正式跑 `/gdoc-highlights-summary` 雙引擎對照**

今天最有感不是上面兩條觀察，是中午跑了一次 `/gdoc-highlights-summary` skill。把高山症訪談 Google Doc 上自己畫的 21 條螢光線**同時餵給 Claude Sonnet 與 Codex CLI gpt-5**——同 prompt、同素材、兩引擎並列產出。Roam 上開了 `[[May 1st, 2026]]` 的對照頁，左欄 Claude Sonnet 版、右欄 Codex gpt-5 版，肉眼挑哪邊留。後來甚至順手又跑了一次 EMM 997 D-Dimer Podcast 14 條畫線的雙引擎對照，連續 A/B 兩次。

這比看 Boris Cherny 講概念更有「我做了」的價值。**以前都是「Claude 寫一遍就結束」沒有 baseline，今天同一份素材 + 同 prompt 在兩模型上 A/B 對照，挑工具就有更直接的判斷依據**：哪個語氣比較貼自己、哪個摘要密度合適、哪個會偷加 LLM 自己的延伸（俗稱發散）、哪個更乖地照素材原文重組。一次跑完心裡會有「下次給這類任務 default 用 X、複雜度高才換 Y」的直覺，這是看再多別人的截圖都建立不起來的。

連起來看，三件事其實是同一件事的三個面向：**Claude Code 工具生態正從「單機助手」演化成「可組合的 AI workflow 平台」**——Boris Cherny 講為什麼（loop、schedule、skill、控制點四個基本元件），Open Design 示範怎麼解封閉（不綁 Anthropic、本地部署、可攜 skill 檔），自己今天的雙引擎對照在實踐怎麼用（同任務跨模型對照建立工具感）。三條線指向同一個結論：**未來的關鍵能力不是會用某一家 LLM，而是會把 workflow 拆成可組合的元件 + 會挑工具**。
